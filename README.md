# Contoso Coffee Machine Ordering System - High-Fidelity Prototype

An interactive canvas application built using **Microsoft Power Apps** representing a product selection and ordering ecosystem for "Contoso Coffee". This project serves as a Business Analysis artifact demonstrating high-fidelity prototyping, UI/UX requirements elicitation, and catalog data structuring.

## 📋 Business Case & Problem Statement
Contoso Coffee sales representatives and retail customers lacked a centralized, interactive platform to seamlessly filter, compare, and order from their extensive catalog of residential and commercial brewing hardware. 

As a Business Analyst, I initiated this prototyping project to:
1. **Bridge the Requirement Gap:** Create a visual alignment tool for stakeholders to agree on user flows.
2. **Segment B2B vs B2C Journeys:** Provide distinct user pathways for home consumers vs commercial accounts.
3. **Validate Comparison Logic:** Demonstrate how complex attributes (pricing, capacity, power tiers) could be evaluated side-by-side by a user.

---

## 🛠️ Key Features & User Interface

### 1. Dynamic Catalog Segmentation
The application splits inventory into 4 core functional categories through an intuitive left-rail navigation menu:
* **At Home Coffee Makers:** Consumer-grade automated drip machines (e.g., *Travel Brew*, *Precision Brew* models ranging $349–$449).
* **At Home Espresso Machines:** Semi-professional counter units (e.g., *Crema Café*, *Crema Quatro* variants ranging $999–$1,399).
* **Commercial Coffee Makers:** Office and catering high-volume drip units (e.g., *Airpot* lines).
* **Commercial Espresso Machines:** Professional multi-group industrial espresso stations (e.g., *Cafe S-Series* & *A-Series* ranging up to $23,900).

### 2. Product Evaluation & Feature Comparison Matrix
* Checkbox hooks (`Compare`) attached to each product card allow users to selectively flag individual hardware models.
* Centralized action buttons (`Compare` and `Clear Selection`) drive data filtering variables to parse comparison views.

## 📐 Business Analysis Artifacts Demonstrated
* **User Story Validation:** Utilized this functional mock-up to map out real user stories regarding e-commerce browsing behavior.
* **Data Architecture Mapping:** Organized product pricing rules, asset naming conventions, and categorization tags into flattened collections mimicking a production database model.
* **Agile Feedback Loops:** Used the prototype to capture instantaneous UX feedback, drastically reducing the scoping timeline for development teams.
